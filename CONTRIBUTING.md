# دليل المساهمة في GreenScope AI

شكراً لاهتمامك بالمساهمة في مشروع **GreenScope AI**. يهدف هذا الدليل إلى مساعدتك في فهم آلية المساهمة والمعايير المتبعة.

## كيف تساهم؟

1. قم بعمل **Fork** للمستودع.
2. أنشئ فرعاً جديداً لميزتك أو إصلاحك:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. نفّذ تغييراتك مع الالتزام بمعايير الكود.
4. تأكد من نجاح فحص الأنواع والبناء:
   ```bash
   pnpm run typecheck
   pnpm run build
   ```
5. ارفع التغييرات وافتح **Pull Request** يصف ما قمت به وسبب التغيير.

## معايير الكود

- استخدم **TypeScript** في جميع الملفات الجديدة.
- اتبع البنية الحالية للمشروع ولا تنشئ ملفات بدون داعٍ.
- التزم بنمط التسمية المستخدم في كل حزمة.
- تجنّب إضافة بيانات وهمية أو placeholders داخل الكود الإنتاجي.

## بنية الفروع

- `main` — الفرع الإنتاجي المستقر.
- `feature/*` — الميزات الجديدة.
- `fix/*` — إصلاحات الأخطاء.
- `chore/*` — مهام صيانة وتحديثات تبعيات.

## رسائل الـ Commit

نفضّل اتباع نمط [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: إضافة شاشة تحليل النباتات
fix: تصحيح خطأ في استدعاء API الصحة
chore: تحديث تبعيات pnpm
docs: تحديث ملف README
```

## الإبلاغ عن المشاكل

افتح **Issue** جديداً مع وصف واضح يحتوي على:

- خطوات إعادة إنتاج المشكلة.
- السلوك المتوقع والسلوك الفعلي.
- لقطات شاشة أو سجلات إن أمكن.
